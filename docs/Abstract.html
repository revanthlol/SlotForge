01 — Abstract

Project Abstract

SlotForge is a web-based institutional timetabling platform designed to automate the construction, validation, editing, and publication of academic schedules. Manual timetable creation requires repeated trial-and-error coordination between sections, teachers, subjects, rooms, weekly hour requirements, lab blocks, and institutional rules. As the number of resources increases, the scheduling space becomes combinatorial, making manual approaches slow, error-prone, and difficult to revise when a constraint changes.

The system models timetable generation as a Constraint Satisfaction and Constraint Optimisation problem. It uses Google OR-Tools CP-SAT to assign subjects, teachers, rooms, sections, days, and periods while enforcing hard constraints such as teacher non-overlap, room non-overlap, section non-overlap, room capacity, teacher availability, configured weekly subject hours, and multi-period lab sessions. The solver also supports soft optimisation goals including teacher gap reduction, daily load balancing, resource utilisation, and preference satisfaction.

SlotForge supports fixed weekday scheduling and day-order scheduling. Administrators configure cycle length, periods per day, teachers, rooms, subjects, section sizes, teacher-subject qualifications, and section-specific subject mappings. Section-subject mapping ensures that a subject is generated only for the sections that actually take it, preventing subjects from being incorrectly assigned to unrelated sections.

The administrative interface includes searchable resource management, customizable subject color coding, guided onboarding, organization switching, editable draft timetables, version history, export tools, and an interactive relationship canvas. The same saved subject color is used across the subject table, timetable grid, all-class list, and canvas graph so generated schedules remain visually consistent and easy to distinguish.

The backend uses FastAPI, SQLAlchemy, Alembic, Pydantic, PostgreSQL, and Supabase Auth. Every tenant-scoped record is associated with an organization, API queries are filtered by the authenticated user's active organization, role-based access control protects mutating endpoints, and audit logs record administrative changes. Timetable generations are stored as immutable versions that can be reviewed, published, archived, rolled back, and exported as PDF, Excel, or CSV.

02 — Implemented Features

Core Feature Set

Constraint Solver

CP-SAT timetable generation with hard constraints, soft scoring, lab-block sessions, fixed weekday and day-order cycles, and infeasibility diagnostics.

Resource Management

Teachers, rooms, subjects, sections, capacities, weekly hours, session lengths, and teacher-subject qualifications.

Curriculum Mapping

Section-specific subject inclusion with optional fixed teacher assignment, preventing unrelated subjects from appearing in generated section timetables.

Timetable Views

Section, teacher, and room perspectives with board mode, all-class list, draft editing, manual insertion, and conflict validation.

Color System

Preset palette and custom color picker with consistent subject colors across resources, timetable cards, list rows, and canvas links.

Canvas View

Interactive graph linking sections, subjects, teachers, and rooms with focus mode, version selector, summary metrics, legend, and selected-node details.

Versioning and Exports

Draft, published, and archived timetable versions with rollback support, version history, stored scores, and PDF, Excel, and CSV exports.

Onboarding and Organizations

Guided setup checklist, new organization creation, organization memberships, and active organization switching with isolated resources.

05 — Engineering Significance

Engineering Significance

Formal optimisation model

The system transforms timetable generation into a CP-SAT model that validates resource, teacher, room, section, hour, and lab-block constraints simultaneously.

Actionable infeasibility reporting

When a solution is impossible, the solver reports useful causes such as capacity shortage, room mismatch, missing teacher qualification, or unavailable legal start periods.

Versioned operational workflow

Generated schedules become immutable versions that administrators can compare, publish, archive, roll back, edit in draft form, and export.

Multi-tenant application design

Organization-scoped records, memberships, active organization switching, RBAC, and audit logs make the system suitable for multiple independent institutions.

Consistent visual system

Subject colors and readable Material-style typography are shared across resource pages, timetable grids, all-class lists, and relationship graphs.

Complete administrative experience

Onboarding, setup guidance, resource CRUD, curriculum mapping, solver execution, timetable review, canvas inspection, and export tools are integrated into one interface.
